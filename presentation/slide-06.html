Where To Start

AI 도입은 전면 자동화보다,
작은 범위의 구조화된 자동화부터 시작하는 것이 현실적입니다.

OpenSDLC는 “처음부터 끝까지 AI가 다 한다”는 식보다, 조직이 감당 가능한 수준에서 단계적으로 자동화를 넓혀 가는 방법을 제안합니다.

Recommended Adoption Path

요구사항 구조화
→
테스트 설계 선행
→
구현 보고
→
실행 검증
→
반복 피드백
Phase 01

기준을 먼저 구조화

요구사항 정리와 테스트 설계처럼, 조직의 기준을 더 명확하게 만드는 단계부터 자동화합니다.

Phase 02

산출물과 증거를 연결

구현 결과를 artifact로 남기고, 실행 검증과 evidence를 연결해 AI 산출물의 설명 가능성을 높입니다.

Phase 03

반복 루프까지 확장

feedback 기반 Spiral Iteration까지 연결해, 반복적 개선 루프를 운영 프로세스 안에 정착시킵니다.

어디서부터

  • 요구사항 정리
  • 테스트 기준 설계
  • 산출물 추적과 보고 체계

어떻게

  • 처음에는 일부 단계만 자동화
  • 사람 승인과 검증 게이트는 유지
  • 운영 안정성이 확인되면 범위를 점진 확대
